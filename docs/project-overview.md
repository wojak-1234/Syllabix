> **서비스 한 줄 정의**

## 0. v2 → v3 구조 변경 이유

### 변경 전 (v2) — 문제 있던 구조
- 수요 예측 기반으로 AI가 강의 주제를 추천 → 강사가 제작
- 학생이 직접 AI 커리큘럼 생성 (목표 입력 → 로드맵 자동 설계)

### 왜 바꿨나
```
수요 예측 → 강사 제작 구조의 현실적 한계:
  - 수요가 있다고 강사가 바로 강의를 만들 수 없음
  - 공급자(강사)가 수요를 따라가지 못하는 속도 문제
  - 강사 자율성 침해 — 강사는 자기가 잘 아는 걸 가르쳐야 함

학생 커리큘럼 자동 생성의 한계:
  - 강사 없이 AI가 짠 커리큘럼은 실제 강의와 연결이 약함
  - 학생이 커리큘럼을 스스로 관리하는 부담 발생
```

### 변경 후 (v3) — 확정 구조
```
Teacher: 커리큘럼 설계 + AI 보조 고도화 + 검토·확정
Student: 목표 입력 → 커리큘럼 추천 → Join → 실습
```

> 강사가 설계하고, AI가 보조하고, 학생이 선택한다.
> 공급과 수요의 속도 문제가 자연스럽게 해결된다.

## 1. 핵심 구조

### 전체 흐름

```
[Teacher]                          [Student]
─────────────────────────────────────────────────────
커리큘럼 설계 (AI 보조)
  ↓
강좌(Lecture) 구성
  ↓
퀴즈 + 코딩테스트 출제
  ↓
커리큘럼 검토 · 확정 · 공개
                                   ↓
                              목표 입력
                                   ↓
                         AI가 맞는 커리큘럼 추천
                                   ↓
                            커리큘럼 Join
                                   ↓
                         강좌 수강 → IDE 실습
                                   ↓
                      퀴즈 / 코딩테스트 → 이해도 점검
                                   ↓
                    실패 시 → 오답노트 자동 생성 (보충)
```

## 2. Teacher 영역

### T-01. 커리큘럼 설계 (AI 고도화) ★ 핵심

강사가 직접 커리큘럼 구조를 설계하되, AI가 전 과정을 보조한다.

#### 설계 흐름

```
Step 1. Series(커리큘럼) 생성
  - 제목, 대상 수준, 목표 수강자 설정
  - Draft 자동저장 (언제든 이어서 작업 가능)

Step 2. Lecture(강좌) 추가
  - 여러 강좌를 순서대로 추가 → 하나의 Series 구성
  - 드래그로 순서 조정 가능

Step 3. 각 강좌 콘텐츠 구성
  ├── 학습 목표 (필수 — RAG 앵커 역할)
  ├── 핵심 개념 태그 (필수 — Blind Point 매칭 기준)
  ├── 첨부파일 (선택 — PDF, 슬라이드)
  ├── 퀴즈 생성 (선택 — 선지 + 문제 직접 타이핑)
  └── 코딩테스트 생성 (선택 — AI 생성 or 직접 입력)

Step 4. AI 보조 고도화
  - 학습 목표 입력 시 → AI가 강좌 구성 초안 제안
  - 개념 태그 자동 추천
  - 코딩테스트 AI 생성 (원하는 스타일 입력 → 다층검증 → 문제 출제)
  - 퀴즈 난이도 자동 분류

Step 5. 검토 · 확정 · 공개
  - 강사가 전체 커리큘럼 미리보기 + 검토
  - 공개 설정 (전체 공개 / 링크 공개 / 비공개)
  - 게시 시 RAG 파이프라인 자동 구축 (임베딩 생성 → pgvector 저장)
```

#### 코딩테스트 생성 방식

```
AI 생성 모드:
  강사가 원하는 스타일 입력
    → Claude가 문제 초안 생성
    → 다층 검증 (문법 오류 / 난이도 / 정답 가능성)
    → 강사 검토 후 확정

직접 입력 모드:
  문제 + 입력/출력 예시 + 정답 코드 + 채점 기준 수동 작성
```

### T-02. 커리큘럼 검토 대시보드

공개 후 운영 중인 커리큘럼을 모니터링한다.

```
├── 수강생 현황 (Join 인원, 진도율)
├── 강좌별 이해도 히트맵
│     "클로저 강좌에서 수강생 67% 코딩테스트 실패"
├── 개념별 오답률 랭킹
├── AI 강의 개선 제안
│     "이 개념에 대한 추가 설명이 필요합니다"
└── 이탈 위험 수강생 감지 알림
```

#### Blind Point 매칭 구조

```
강의 텍스트 → 임베딩 (RAG)
학생 오답   → 개념 태깅
                ↓
          벡터 유사도 매칭
                ↓
"클로저 설명 구간 ↔ 수강생 오답 패턴" 연결
                ↓
강사에게 인사이트 제공:
  "이 구간 설명이 미흡하거나 보충 설명이 필요합니다"
```

## 3. Student 영역

### S-01. 목표 입력 → 커리큘럼 추천

학생은 커리큘럼을 직접 만들지 않는다.
**목표를 입력하면 AI가 맞는 커리큘럼을 추천해준다.**

```
목표 입력: "3개월 안에 React로 포트폴리오 완성"
              ↓
      현재 수준 진단 (온보딩 5문제)
              ↓
    AI가 등록된 커리큘럼 중 매칭
      - 난이도 적합성
      - 목표 달성 가능성
      - 예상 소요 기간
              ↓
    추천 커리큘럼 목록 제시
      (강사명 / 총 강좌 수 / 예상 기간 / 수강생 수)
              ↓
       학생이 선택 → Join
```

---

### S-02. 강좌 수강 + 이해도 점검

```
강좌 콘텐츠 학습
        ↓
   이해도 점검 (두 가지)
   ┌─────────────────┐
   │  퀴즈           │  → 개념 이해 확인
   │  코딩테스트     │  → 실제 구현 능력 확인
   └─────────────────┘
        ↓
    성공 → 다음 강좌
    실패 → 오답노트 생성 + 보충
```

---

### S-03. Dynamic IDE + AI 페어 프로그래머 ★★ 최핵심

**원칙: AI는 답을 주지 않는다**

#### 제출 구조

```
문제 제시
    ↓
코드 작성 (Monaco Editor)
    ↓
제출 (최대 3회)
    ↓
매 제출마다:
  - 코드 스냅샷 저장
  - 에러 타입 분류
  - AI 소크라테스 힌트 제공
```

#### AI 힌트 단계

```
1차 실패 → 방향 힌트
  "어떤 부분이 문제일 것 같아?"
  "에러 메시지 3번째 줄이 뭘 말하는 것 같아?"

2차 실패 → 개념 힌트
  "for문 끝에 어떤 기호가 필요하지?"
  "의존성 배열에 뭐 넣었는지 확인해봤어?"

3차 실패 → 구조 힌트
  "콜론(:)을 확인해보는 건 어떨까?"
  "useEffect 두 번째 인자가 하는 역할을 떠올려봐"

※ 직접 코드 수정은 절대 하지 않음
```

#### 에러 타입 → 개념 태깅

```
SyntaxError   → 문법 개념 취약
TypeError     → 타입 개념 취약
LogicError    → 알고리즘 사고 취약
RuntimeError  → 실행 흐름 이해 취약
```

### S-04. 오답노트 자동 생성 (보충)

#### 트리거

```
퀴즈 오답 OR 코딩테스트 3회 실패
  → 오답노트 자동 생성
```

#### 오답노트 구조

```
┌────────────────────────────────┐
│ 문제명    : 반복문 출력 문제   │
│ 유형      : 코딩테스트         │
│ 에러 타입 : SyntaxError        │
│ 틀린 이유 : 콜론(:) 누락       │
│ 관련 개념 : for문 문법         │
│ 시도 횟수 : 3회                │
│ 코드 스냅샷: [1차][2차][3차]   │
└────────────────────────────────┘
```

#### 보충 학습 연결

```
오답노트 생성
    ↓
관련 개념 RAG 검색
    ↓
해당 강좌의 관련 구간 자동 연결
    ↓
"이 개념을 다시 확인해보세요" + 보충 링크 제공
    ↓
보충 완료 후 → 재도전 유도
```

#### 커리큘럼 자동 반영

```
동일 개념 2회+ 오답 누적
  → 다음 강좌 진입 전 보충 강좌 자동 삽입
  → 학생 알림: "보강 학습이 추가됐어요"
  → 강사 알림: "N명이 이 개념에서 반복 오답 중"
```

## 5. AI 기술 활용 맵

| 기능 | AI 패턴 | 역할 |
|------|---------|------|
| 커리큘럼 추천 | Structured Output + RAG | 목표·수준 기반 매칭 |
| 강좌 초안 제안 | Multi-step Chain | 목표→구성→문제 순차 생성 |
| AI 페어 프로그래머 | Multi-turn Agent + Streaming | 소크라테스 힌트 대화 |
| 오답 개념 태깅 | Structured Output | 에러→개념 분류 JSON |
| Blind Point 탐지 | LangChain Agent | 오답 패턴 분석 자율 실행 |
| 강의-오답 매칭 | RAG (pgvector) | 강의 구간 ↔ 오답 개념 연결 |
| 코딩테스트 생성 | Multi-step Chain + 다층검증 | 문제→검증→정제 |

## 7. MVP 우선순위
