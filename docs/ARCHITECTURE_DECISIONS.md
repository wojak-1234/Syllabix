## Phase 1 — 아이디어 발굴 : 공모전 주제 해석

### 배경
공모전 주제: **"AI 활용 차세대 교육 솔루션"**  
심사 기준: 기술적 완성도 / AI 활용 능력 / 기획력 및 실무 접합성 / 창의성

### 초기 질문
> "수상권에 들 법한, 기존 서비스에는 없는 아이디어를 제시해줘"

### AI 제안 (6가지 아이디어)
AI가 제안한 후보군:

| 아이디어 | 설명 |
|---|---|
| AI 강의 미러링 & 질 분석 코치 | 강의 녹화본 분석 → 강사 피드백 |
| AI 학습 공백 레이더 | 행동 데이터로 "모르는 줄 모르는" 개념 탐지 |
| AI 교육 운영 자동화 | 문의·행정 반복 업무 자동화 |
| 실시간 AI 이해도 체크 레이어 | 라이브 강의 중 이해도 실시간 감지 |
| **AI 커리큘럼 역설계 엔진** | 목표에서 커리큘럼을 거꾸로 설계 |
| AI 피어러닝 다이나믹 매칭 | 강점이 다른 학습자를 AI가 매칭 |

### 의사결정 1 — 방향 후보 선택
- **채택**: AI 커리큘럼 역설계 엔진 + AI 학습 공백 레이더 (Blind Point)
- **기각**: 강의 미러링, 운영 자동화, 피어러닝 매칭
- **기각 이유**:
  - 운영 자동화는 교육 솔루션보다 CS 툴에 가까움
  - 피어러닝 매칭은 콘텐츠 없이 매칭만으로는 약함
  - 강의 미러링은 강사 수가 적은 초기 서비스에 데이터 부족

> **핵심 인사이트**: "목표에서 역산하는 커리큘럼"과 "모르는 걸 AI가 먼저 발견"이라는 두 축이 이 프로젝트의 정체성이 됨

## Phase 2 — 기능 정의 : 첫 번째 기획안

### 배경
Phase 1에서 선택한 두 축을 기반으로 구체적인 기능 목록을 정의

### 초기 기획 (사람이 먼저 제시)

**Student Side**
- AI 커스텀 커리큘럼 생성
- Frequent Blunder List (오답노트)

**Teacher Side**

- Blind Point 확인
- 학생 실패 포인트 확인

### AI 구현 방안 제안
AI가 각 기능별 구현 방식, 기술 스택, 데이터 흐름을 제안

### 의사결정 2 — 기능별 구현 방식 채택
| 기능 | AI 제안 | 채택 여부 | 이유 |
|---|---|---|---|
| 커리큘럼 생성 | Structured Output + RAG | ✅ 채택 | JSON 강제 반환으로 파싱 오류 없음 |
| 블런더 리스트 | 오답 클러스터링 + Claude 분석 | ✅ 채택 | 행동 데이터 기반 패턴 추출 |
| Code Reviewer | Multi-step Chain (3단계) | ✅ 채택 | 단일 호출보다 맥락 반영 정밀도 높음 |
| Blind Point | LangChain Agent | ✅ 채택 | 자율 도구 호출 구조가 핵심 차별화 |

## Phase 3 — AI 기술 다양화 : Agentic AI 도입

### 배경
> "AI들을 단순 chatbot 형태가 아닌 다른 여러 방식으로 활용하고 싶어"

### AI가 제안한 5가지 패턴
1. **Structured Output** — JSON 생성 엔진
2. **RAG** — 강의 콘텐츠 기반 답변
3. **LangChain Agent** — 자율 판단·행동 (Blind Point 탐지)
4. **Multi-step Chain (LCEL)** — 코드 리뷰 3단 파이프라인
5. **Streaming** — 실시간 피드백 UX

### 의사결정 3 — Agentic AI 구조 채택
- **채택**: Agent (Blind Point 탐지) + Multi-step Chain (코드 리뷰) + Streaming (UX)
- **판단**: 단순 Claude 호출이 아닌, AI가 스스로 도구를 선택해 순서대로 실행하는 구조가 심사 차별화 포인트

**Blind Point Agent 도구 목록 확정**
```
Tool 1: get_behavior_logs       — 행동 로그 조회
Tool 2: get_wrong_answer_patterns — 오답 패턴 분석
Tool 3: update_blunder_book     — 블런더북 업데이트
Tool 4: trigger_curriculum_reset — 커리큘럼 리셋
Tool 5: notify_teacher          — 강사 알림 발송
```

> **핵심 전환점**: "AI가 대답한다"에서 "AI가 판단하고 행동한다"로 개념 전환

## Phase 4 — 방향 전환 1 : 범용 플랫폼 시도

### 배경
> "코딩으로만 주제를 국한하면 좀 그래. 제빵, 위험물관리사 같은 포괄적인 주제를 다룰 수 있게 방향성을 변경한다면?"

### AI 제안 — 범용 플랫폼 메뉴 재설계

- 추가: AI 개념 탐험가 (Knowledge Graph), AI 모의시험 생성기, 실전 시뮬레이터, AI 학습 저널, 커뮤니티 Q&A

### 의사결정 4 — 범용 전환 시도 후 재검토 예정
- **임시 채택**: 범용 방향 탐색
- **보류 이유**: IDE와 코드 리뷰어를 유지하면 어정쩡해짐. 범용으로 가면 실습 메커니즘 전체를 교체해야 함

## Phase 5 — UX 진단 : 기존 기획의 문제점 발견

### 배경
> "내 UX는 뭐가 문제인 것 같아?"

### 기존 기획 흐름 재검토
**Teacher**: 수요확인 → 강의제작 → 업로드 → Blind Point 수신  
**Student**: AI 커리큘럼 생성 → 문제 제시 → Dynamic IDE

### AI가 발견한 UX 문제 4가지

#### 문제 1 — Teacher: 수요 확인과 강의 제작 사이 단절 `치명적`
- **해결**: 수요 키워드 클릭 → AI 강의 초안 원클릭 생성 → 강사는 검토만

#### 문제 2 — Student: 커리큘럼 생성 후 "그래서 뭐 해?" 공백
- **해결**: 커리큘럼 주차 클릭 → 관련 강의 자동 추천 → 퀴즈 → IDE 원스텝

#### 문제 3 — Blind Point가 강사에게만 가고 학생에게 안 감
- **해결**: Blind Point → 강사(강의 개선) + 학생(즉시 보강) 동시 발송

#### 문제 4 — 진단 없이 커리큘럼 생성
- **해결**: 온보딩 진단 5문제 → 수준 파악 후 커리큘럼 생성

## Phase 6 — 방향 전환 2 : 코딩 특화 확정

### 배경
> "코딩에만 국한한다면 어떤 기능을 넣으면 좋을까? UX를 매끄럽게 다듬고 싶어"

### AI 제안 — 코딩 특화 추가 기능
- **AI 페어 프로그래머**: 소크라테스식 질문으로 자력 해결 유도
- **시도 히스토리 타임라인**: 코드 제출 과정 저장 → Blind Point 정밀도 향상
- **학습 패턴 뱃지**: 힌트 없이 해결, 클린코드 달성 등
- **AI 다중 풀이 비교**: 버블정렬 성공 후 퀵정렬·머지정렬 비교 제시

### 의사결정 6 — 코딩 특화 최종 확정
- **채택**: 코딩 특화 유지
- **채택 이유**:
  - IDE는 코드 실행 환경이 있어야 의미 있음
  - "답을 주지 않는 AI 페어 프로그래머"가 플랫폼 핵심 가치와 정확히 일치
  - 타겟이 명확할수록 심사 기준 "실무 접합성"에서 유리

**최종 확정 학습 루프**
```
진단 테스트 → AI 커리큘럼 생성 → 강의 추천·수강
→ 개념 확인 퀴즈 → Dynamic IDE (AI 페어 프로그래머)
→ Blind Point 탐지 → 자동 복귀·재도전 → 다음 주차
```
